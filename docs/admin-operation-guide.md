# 夜郎秘境后台管理与字段配置说明

## 使用原则

后台用于维护双语内容、保护坐标、用户权限、积分规则、审核和接口配置。删除操作不可恢复；删除媒体时系统同时删除 OSS 对象。所有用户生成内容应由管理员最终确认，AI 初审只能作为辅助建议。

## 菜单操作

### 秘境管理

用于新增、编辑、审核和删除景点。景点通过审核且启用后才会出现在小程序。

| 字段 | 配置说明 |
| --- | --- |
| 中文/英文名称、简介、介绍 | 小程序中英文展示内容；英文不应只是机器直译，应保留地名和风险提示。 |
| 市州、区县、纬度、经度 | 主景点位置。经纬度使用 WGS-84 十进制度，纬度在前、经度在后；保存前检查是否落在贵州或合理区域。 |
| 标签 | 关联首页筛选条件，可多选。 |
| 可见级别 | `public` 公开、`member` 会员、`protected` 保护、`secret` 守护；保护级别需配合坐标保护使用。 |
| 秘境等级 | 从通关设置已有等级选择，决定地图标识颜色和每次打卡积分。 |
| 解锁积分 | 用户探秘积分达到该值后才会获得景点数据。 |
| 打卡半径 | 米。用于后台自动判断用户位置与景点距离，建议按入口、停车点或安全观察点设置。 |
| 河流与上游坐标 | 填写河流名称和上游经纬度后，详情页可请求上游天气/预警。 |
| 子景点坐标 | 可维护多个坐标及备注，每个点可单独选择是否获取天气；删除前应确认不再被路线或内容引用。 |
| 审核状态 | `draft` 草稿、`pending` 待审、`approved` 通过、`rejected` 拒绝。仅通过且启用的景点对小程序可见。 |
| 图片/视频 | 管理员上传图片不超过 2MB、视频不超过 8MB，文件存 OSS；可设置封面、排序、停用或删除。 |

### 标签管理

维护中英文名称、图标、排序、启用状态。排序数字不可重复，避免首页筛选顺序不稳定；删除前应先解除与景点的关联。

### 用户管理

用户由微信登录自动创建，也可后台新增。OpenID 是唯一身份，不可与其他用户重复。昵称、头像由小程序资料保存同步；没有头像时显示系统默认头像。

| 字段 | 配置说明 |
| --- | --- |
| 探秘积分 | 决定可解锁景点与会员自动升级。 |
| 打卡数 | 自动成功打卡后增加。 |
| 贡献数、环保信用 | 当前为统计参考字段，后台不可直接编辑。 |
| 会员、启用 | 会员影响会员可见景点；停用后用户不能继续使用相关功能。 |
| 上传图片、上传视频、留言游记、打卡通关 | 用户功能开关，小程序会按权限隐藏对应入口。 |

### 通关设置

每条规则代表一个可编辑且不可重复的等级编号。

| 字段 | 配置说明 |
| --- | --- |
| 等级、中文/英文名称 | 地图等级和后台展示名称。 |
| 标识颜色 | 地图自定义图钉颜色，使用 `#RRGGBB`。 |
| 解锁所需积分 | 用户积分达到后可解锁该等级关联的景点。 |
| 每次打卡积分 | 用户成功打卡该等级景点时增加的探秘积分。 |
| 会员要求、解锁权益 | 控制会员门槛和后台/小程序说明。 |

### 会员管理

维护套餐名称、价格、周期、双语权益、启用状态和升级积分。系统会根据用户探秘积分自动匹配满足条件的会员等级；变更门槛前应确认现有用户影响。

### 打卡审核

列表展示用户、景点、自动判定状态、距离与时间。GPS 自动判断是主要结果；管理员可按景点、用户、状态、时间筛选并复核异常记录。打卡成功会按景点等级累计积分。

### 游记留言

游记和留言默认待审核。审核详情可逐个通过、删除或拒绝图片/视频；拒绝或删除会同步清理 OSS 文件。通过的游记、留言和媒体才会渲染到小程序详情页。可点“AI 初审”获得文字与媒体风险提示，但必须由管理员做最终操作。

### 衣食住行

关联景点，类别为衣、食、住、行。维护双语名称、双语简介、地区、地址、联系方式、价格级别、推荐等级、启用状态和媒体。图片/视频同样限制 2MB/8MB 并保存在 OSS。

### 接口管理

- 天气接口：配置和风 Host、项目 ID、凭据 ID、私钥和 JWT 有效期；用“测试和风天气”验证。测试失败时检查 Host、项目/凭据匹配和公私钥配对。
- 大模型接口：配置服务商、OpenAI 兼容 API Base、模型名与 API Key；用“测试大模型连接”验证。若模型支持图片输入，可启用“图片视觉初审”，游记/留言的图片会随 AI 初审请求提交；视频仍需人工查看。AI 管理助手读取此配置。
- 河流洪水接口：预留第三方水文数据配置；未配置时以天气预警和人工风险提示兜底。
- 小程序数据时间：启用后设置北京时间开始/结束小时，超出时段的小程序不请求业务数据。
- 对象存储：配置 OSS endpoint、region、bucket、访问域名；AccessKey 仅存在服务器 `.env`，不写入后台数据库。使用测试连接验证权限。

## AI 管理助手知识与审核

右下角 AI 浮动按钮在已登录后台的所有页面显示，数字角标汇总待审核打卡、游记、留言和媒体。它可：

1. 根据本文件和小程序设计说明回答菜单操作与字段配置。
2. 根据管理员输入的景点资料生成中英文简介草稿。
3. 说明坐标采集、坐标核验和保护坐标处理方法。
4. 对游记/留言文本和媒体链接给出“通过、人工复核、拒绝”的初步建议。

如果模型不支持视觉，AI 必须明确不能识别图片/视频本身；媒体审核仍应由管理员人工查看。AI 不直接修改数据库、不直接通过或拒绝内容。

## 微信通知可行性

可以使用微信小程序订阅消息提醒管理员有待审核内容，但有前置条件：管理员的微信用户必须主动订阅对应模板，后台需保存该管理员 OpenID、模板 ID，并在微信公众平台配置服务类目与模板。未授权时微信不会投递；因此后台浮动角标和审核列表必须保留为可靠的站内提醒。建议后续新增“通知管理”配置、管理员接收人管理和发送日志，再接入订阅消息。
