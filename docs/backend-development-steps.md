# 贵州秘境后台开发步骤

## 目标

后台部署在阿里云服务器，第一阶段先支撑小程序 MVP 和管理后台 MVP：

- 小程序首页贵州地图秘境点位查询
- 标签筛选与中英文内容返回
- 秘境详情查询
- 管理后台维护秘境、标签、坐标可见级别和审核状态
- 管理后台账号登录、JWT 鉴权和基础角色
- 管理小程序注册用户、会员状态、探索等级和账号启停
- 管理 L0-L5 探索通关设置，包括打卡数、贡献数、环保信用、会员要求和解锁权益
- 管理会员套餐、用户会员记录和打卡审核
- 管理秘境图片、用户游记、用户留言和衣食住行推荐
- 为后续微信登录、会员、打卡、AI 推荐、RAG 知识库预留接口边界

## 技术选型

- API 服务：FastAPI
- 数据库：MySQL 8.x
- 缓存：Redis 7.x
- ORM：SQLAlchemy
- 迁移：Alembic
- 部署：阿里云 ECS + Docker Compose
- 反向代理：Nginx
- 进程：Uvicorn / Gunicorn
- 对象存储：阿里云 OSS，后续用于景点图片、打卡图片和内容素材

## 阿里云部署架构

```text
微信小程序 / 管理后台
        |
     HTTPS
        |
阿里云 SLB 或 ECS Nginx
        |
FastAPI 后台服务
   |          |
 MySQL      Redis
   |
 OSS / AI 服务 / 地图服务
```

## 开发阶段

### 第 1 阶段：工程初始化

1. 创建 `backend` 工程目录。
2. 配置 FastAPI 应用入口、版本化 API 路由、健康检查接口。
3. 准备 `.env.example`、`requirements.txt`、`Dockerfile`、`docker-compose.yml`。
4. 统一配置读取方式，预留 MySQL、Redis、JWT、OSS、AI、地图服务密钥。

### 第 2 阶段：核心数据模型

1. 创建秘境表：中文名称、英文名称、中文介绍、英文介绍、经纬度、市州、区县、推荐等级、可见级别、审核状态。
2. 创建标签表：中文标签、英文标签、图标、排序、启用状态。
3. 创建秘境标签关联表。
4. 创建用户、收藏、打卡、会员、AI 问答记录等后续表的模型骨架。

### 第 3 阶段：小程序 API

1. `GET /api/v1/tags`：返回启用标签，支持 `lang=zh-CN/en-US`。
2. `GET /api/v1/spots/map`：按标签、语言、用户等级返回地图点位。
3. `GET /api/v1/spots/{spot_id}`：返回秘境详情。
4. 点位坐标按可见级别处理：公开点位返回精确坐标，保护点位返回模糊坐标或区域中心。

### 第 4 阶段：管理后台 API

1. 秘境列表、创建、编辑、上下架、审核。
2. 标签列表、创建、编辑、排序、启停。
3. 点位经纬度与坐标保护策略维护。
4. 中英文内容维护与审核。
5. 注册用户列表、用户详情、探索等级、会员状态和启停管理。
6. L0-L5 通关规则维护，包括打卡数、贡献数、环保信用、会员要求和解锁权益。
7. 会员套餐维护、用户会员记录查看。
8. 打卡记录审核，通过后同步更新用户打卡数。
9. 秘境图片上传、封面设置和图片停用。
10. 用户游记与留言审核、隐藏、精选。
11. 衣食住行推荐维护，包括装备、美食、住宿和交通推荐。

### 第 5 阶段：认证与权限

1. 管理后台账号密码登录。
2. JWT 鉴权，所有 `/api/v1/admin/*` 管理接口必须带 Bearer Token。
3. 角色权限：超级管理员、内容运营、审核员。
4. 小程序微信登录与用户身份绑定。

### 第 6 阶段：AI 与打卡闭环

1. 建立秘境知识库同步任务。
2. 接入 AI 推荐和问答接口。
3. 记录用户问题、推荐结果和反馈。
4. 实现 GPS + 图片识别 + 人工审核的打卡流程。

### 第 7 阶段：上线准备

1. 在阿里云 ECS 安装 Docker、Docker Compose、Nginx。
2. 配置安全组，仅开放 80、443、SSH 管理端口。
3. 配置域名、HTTPS 证书、Nginx 反向代理。
4. 使用 Docker Compose 启动 API、MySQL、Redis。
5. 执行数据库迁移和初始化标签数据。
6. 配置日志轮转、数据库备份、OSS 权限和监控告警。
